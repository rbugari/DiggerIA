A **Solution** is a workspace dedicated to a specific codebase (e.g., "Marketing DWH" or "Sales SSIS Migration").

1.  **Access the Dashboard:** Click on "Solutions" in the main menu.
2.  **Add New:** Click the "**+ New Solution**" button.
3.  **Configuration:**
    - **Name:** Enter a descriptive name.
    - **Path/URL:** Provide the local folder path containing your code or a Git Repository URL.
4.  **Initialize:** Click "Create". The system will begin the **Ingestion Phase**.

## üï∏Ô∏è 3. Exploring the Knowledge Graph
This is the heart of the platform. You can visualize how data flows through your system.

### 3.1 Perspectives: Architect vs. Engineer
Use the toggle in the top-right corner to change the level of detail:
- **Architect View:** Simplifies the graph to show only Tables and Databases. Ideal for high-level business maps.
- **Engineer View:** Shows every script, transformation, and task. Ideal for debugging and technical audits.

### 3.2 Impact Analysis (The "Blast Radius")
1.  Click the "**Impact Mode**" button (Sparkles icon) in the toolbar.
2.  **Select a Node:** Click on any table or script.
3.  **Result:** The graph will instantly highlight all downstream assets affected if that node changes. Everything else will dim.

### 3.3 Context Isolation (Drill-Down)
1.  In a complex graph, click on a **Package Container** (the large boxes grouping tasks).
2.  The view will **Focus** only on that package, hiding everything else.
3.  **Reset:** Click "**Reset Focus**" in the toolbar to return to the global map.
